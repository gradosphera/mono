### Blago Create Requirement Document

Создавать требования необходимо на основе разработанных по bmad документов или по прямому указанию пользователя. 

Создание требований производится с помощью терминальной утилиты `blago` которая доступна из командной строки. 

Перед началом прочитай файл: `~/.claude/config/blago/config.yaml`

Запомни оттуда: `workspace_base`, `active_workspace_env`, и путь из `workspaces`, который соответствует `active_workspace_env` - назовем его `workspace_path`.

Перед продолжением тебе должен быть известен id проекта или компонента в котором создаётся issue. Их нужно извлечь из файла `project.md` или `component.md`, над которым ведется работа. 

После чего выполняй:

```bash
blago create issue <project_or_component_id> "<title>"
```

По-умолчанию документ создаётся в markdown формате. Если передать опцию то формат можно изменить: `--format markdown|drawio|bpmn`

При редактировании markdown файла ты можешь использовать кодовый блок ```mermaid ``` для диаграмм. 

После завершения процедуры создания/редактирования документ следует синхронизировать с сервером. Для этого используй:

```
blago add <issue_id> && blago push
```
   
В случае возникновения ошибок при синхронизации останавливай работу и сообщай об этом.
