### Create Commit

В репозитории программного кода, в котором реализуется проект, необходимо совершать коммиты по факту перевода задач в статус ON_REVIEW. Мелкие коммиты на каждом полу-шаге не совершаем.

Перед началом прочитай файл: `~/.claude/config/blago/config.yaml`

Запомни оттуда: `username`.

Также найди задачу, по которой совершается команда коммита, и извлеки из неё `id` задачи, мотивацию (зачем делали), техспецификацию (что конкретно изменили). 

После чего, создай коммит с помощью git, используя следующий шаблон:
Сформируй сообщение коммита по строгому правилу:

[<id>][@<username>] <тип>: <действие> — <зачем это нужно>


Правила:
- Тип ОБЯЗАТЕЛЕН: feat, fix, refactor, docs, config
- После тире ОБЯЗАТЕЛЬНО указать зачем (бизнес-причина или техническая необходимость)
- Минимум 10 слов в сумме (тип + действие + зачем)
- Не копируй дифф, не пиши "исправлено", "обновлено" без пояснения зачем

Запрещено:
- Писать только "[123] Исправлено"
- Пропускать часть "— зачем"
- Копировать дифф целиком

Где через [@username] мы фиксируем авторство. А через первый id - идентификатор задачи. Обязательно пиши в квадратных скобках как указано. 

После того, как ты создал коммит - запуш его в ветку dev: `git push`
