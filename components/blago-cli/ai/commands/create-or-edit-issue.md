### Blago Create Issue

Создавать задачи необходимо на основе планируемых или проделанных работ. Задачи должна содержать человеко-понимаемую мотивацию к выполнению задачи и краткое описание того, что сделано за ОДНУ пользовательскую сессию. Также следует использовать описание задачи для техспецификации. 

В задачу мы добавляем все то, что сделано за одну сессию с пользователем, т.е. не создаём множество мелких задач, а объединяем их в одну с одной общей техспецификацией за сессию.  

Создание задач производится с помощью терминальной утилиты `blago` которая доступна из командной строки. Перед началом прочитай файл: `~/.claude/config/blago/config.yaml`

Запомни оттуда: `workspace_base`, `active_workspace_env`, и путь из `workspaces`, который соответствует `active_workspace_env` - назовем его `workspace_path`.

Перед продолжением тебе должен быть известен id компонента в котором создаётся issue. id компонента можно извлечь из файла `component.md` над которым ведется работа. Важно: в проектах мы никогда НЕ создаём issue, создаём только в компонентах. 

После чего выполняй:

```bash
blago create issue <component_id> "<title>" --set-self
```

Где `--set-self` установит меня как исполнителя. 

**После выполнения:** в выводе CLI — строка вида `Создан черновик …: <относительный-путь>` (полный путь строить от `workspace_path`). 

Используй этот путь для редактирования задачи. 

Разрешено автоматически менять title, priority, status задач. Допустимые статусы задач: TODO, IN_PROGRESS, BACKLOG. Статусы DONE и ON_REVIEW допустимы только если прямо указано. 

Допустимые приоритеты: HIGH, URGENT, LOW, MEDIUM
    
Редактирование estimate производится в часах только по прямому указанию.

После завершения итерации редактирования производи синхронизацию с сервером через команды:

```
blago add <issue_id> && blago push
```
   
В случае возникновения ошибок при синхронизации останавливай работу и сообщай об этом.
