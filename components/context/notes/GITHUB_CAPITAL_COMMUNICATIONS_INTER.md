# GitHub: переписка и транскрипции проектов Capital

Кратко, как устроено без привязки capital → chatcoop в коде.

## Зачем пакет `@coopenomics/inter`

В нём только контракт: типы сообщений/транскрипций и токен Nest `INTER_PROJECT_COMMUNICATION_ARTIFACTS`. Capital знает интерфейс порта; chatcoop подставляет реализацию (адаптер к своим репозиториям). Сборка контроллера связывает токен с адаптером через глобальный модуль-мост (`InterCommunicationBridgeModule`).

## Откуда данные

- **Проект и чат:** в реестре ChatCoop у проекта Capital есть привязка `project_hash` → чат Matrix (тип `capital_project`). В норме **один проект = один такой чат** (переписка + созвоны). Слово «комната» в коде — технический id чата в Matrix, не «несколько разных чатов у одного проекта», если вы так не настраивали.
- Текст и голос (как текст после Whisper): история сообщений этого чата в БД, UTC-день по `origin_server_ts`.
- Звонки: транскрипции со статусом **completed**; в GitHub — отдельный файл на каждый завершённый звонок.

## Что пишется в GitHub

В каталоге проекта (тот же `basePath`, что и для `project.md`):

| Путь | Содержимое |
|------|------------|
| `messages/YYYY-MM-DD.md` | Один файл на UTC-день. В теле: `#` дата, название проекта, строка с hash, `## Сообщения:`, затем Matrix room и ленты. Несколько чатов — подзаголовки `###` с лейблом комнаты. |
| `meetings/YYYY-MM-DD_HHmmss_mmm.md` | Одна завершённая транскрипция; в имени — момент окончания звонка (UTC). Индекс в БД по-прежнему по id транскрипции. |

Индекс `github_file_indexes` хранит типы `room_message_day` и `call_transcription`, чтобы не дублировать файлы и знать SHA.

## Инкремент и курсоры

Пуш **не** вызывается при обновлении `project.md`, только из cron (`GitHubSyncSchedulerService` после `syncFromGitHub`).

- **Сообщения:** для пары (кооператив, id чата Matrix) хранится последний учтённый `origin_server_ts`. Обычно у проекта один id. Первый запуск без курсора — максимум «сейчас», без бэкфилла. Дальше в Git пересобираются только UTC-дни с новыми сообщениями; затем курсор обновляется.
- **Транскрипции:** на пару (кооператив, `project_hash`) хранится маркер «уже обработано всё с `ended_at` не позже». Первый раз маркер ставится на последний `ended_at` среди completed (или на «сейчас»), без бэкфилла. Новые файлы — только записи с `ended_at` строго больше маркера и без строки в индексе.

Так при сотнях проектов и большом архиве каждую минуту не перекачивается вся переписка — только дельта по курсорам и затронутым дням.
