# XSD-схема ЕФС-1 (4-ФСС) для валидации

Источник: официальные XSD с сайта СФР (`EFS-1_2024-01-01.xsd` + 7 зависимостей).
Оригинальные файлы лежат в `components/reports-standarts/4ФСС-ЕФС-1/ЕФС-1_2-official/`.

## Что патчилось

Оригинальные схемы используют кириллические URI в namespace'ах
(`http://пф.рф/ЕФС-1/2024-01-01`, `http://пф.рф/ВС/ЕФС/2024-01-01` и т.п.).
libxml2 (через libxmljs2) отклоняет такие URI: *"is not a valid URI"*, и
схема не строится. Поэтому при копировании сюда сделаны две подстановки:

1. **Кириллические host + path → ASCII:**
   - `http://пф.рф/ЕФС-1/2024-01-01`  → `http://ns.efs.ru/EFS-1/2026-01-01`
   - `http://пф.рф/ВС/ЕФС/2024-01-01` → `http://ns.efs.ru/VS/EFS/2026-01-01`
   - `http://пф.рф/ВС/типы/2017-10-23`→ `http://ns.efs.ru/VS/types/2025-01-01`
   - `http://пф.рф/АФ/2018-12-07`     → `http://ns.efs.ru/AF/2025-01-01`
   - `http://пф.рф/АФ/2017-08-21`     → `http://ns.efs.ru/AF-old/2017-08-21`
   - `http://пф.рф/УТ/2017-08-21`     → `http://ns.efs.ru/UT/2025-01-01`
   - `http://пф.рф/УТ/2022-07-14`     → `http://ns.efs.ru/UT/2022-07-14`

2. **Корневой элемент `ЭДПФР` → `ЭДСФР`** (чтобы совпал с текущим форматом
   СФР 2026-01-01, используемым в эталоне КОНТУР-ЭКСТЕРН).

Подстановка дат namespace'ов (2024-01-01 → 2026-01-01, 2017-10-23 → 2025-01-01
и т.п.) сделана, потому что структурно XSD 2024-01-01 полностью совместима
с форматом 2026-01-01 эталона СФР (отличие — только даты в namespace URIs).
Это подтверждено прогоном валидации: эталон ПК "Ромашка" проходит схему без
структурных ошибок.

## Совместная подстановка в XML при валидации

`XsdValidatorService` при валидации XML ЕФС-1 применяет **ту же подмену
namespace'ов в памяти** перед вызовом `xmlDoc.validate()`. На диск / СФР
уходит исходный XML с кириллическими namespace'ами.

## Imports и CWD

libxml2 резолвит `<xs:import schemaLocation="..."/>` относительно текущего
CWD процесса. Поэтому валидатор использует `process.chdir(efs1 dir)` на
время вызова `xmlDoc.validate()` (в try/finally, с восстановлением CWD).

## Список файлов

- `efs1.xsd` — корневая схема ЕФС-1 (patched, targetNs 2026-01-01)
- `efs-types.xsd` — типы блоков ЕФС (ТипСтрахователь и т.п.)
- `vs-types.xsd` — общие типы ВС
- `af-types.xsd` — типы АФ (2025-01-01 после патча)
- `af-old-types.xsd` — внутренний АФ (оригинально 2017-08-21)
- `unif-types.xsd` — унифицированные типы УТ
- `unif-types-v2.xsd` — унифицированные типы УТ v2
- `xmldsig.xsd` — W3C XML Digital Signatures (без изменений)
