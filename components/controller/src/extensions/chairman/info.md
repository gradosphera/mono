## Подключение (backend)

Зачем: позволить пройти установочный набор решений/документов на платформе. Шаг считается завершённым только после принятого решения (событие блокчейна).

Принцип:
- Конфиг расширения хранит ожидаемые hash документов по 6 шагам, флаги выполнения и таймеры (start/expire 30 дней).
- Мутации:
  - `completeChairmanAgendaStep` — создаёт проект свободного решения, генерирует документ, сразу публикует его в блокчейн через `publishProjectOfFreeDecision`, сохраняет hash в конфиге (флаг не ставится).
  - `completeChairmanGeneralMeetStep` — сохраняет hash повестки общего собрания (флаг не ставится).
- События: сервис `onboarding-events` слушает `soviet::newresolved`, сравнивает hash с ожидаемым; при совпадении ставит флаг done и очищает hash, если срок онбординга не истёк.
- Стейт: `getChairmanOnboardingState` отдаёт флаги/таймеры для фронта.
- Автозавершение шагов 1–5 допускается на фронте, если в vars уже есть шапки документов (приоритет vars), но основное подтверждение — по событию hash.
