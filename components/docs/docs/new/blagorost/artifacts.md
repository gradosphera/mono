# Артефакты

**Артефакт** — фрагмент описания работы по Проекту или Компоненту: текст, диаграмма, схема. Артефакты создают участники Авторского коллектива (Автор, Соавторы, Мастер) и Исполнители — чтобы зафиксировать Концепцию, требования и проектные решения. Вместе с описанием Проекта, формулировками задач и текстом Коммитов артефакты формируют **Результат интеллектуальной деятельности (РИД)**, который Компонент сдаёт в кооператив как имущественный паевой взнос.


## Где это и когда вы это видите

В Благоросте есть два места для артефактов:

- **На проекте** — карточка проекта → вкладка **«Артефакты»**. Здесь живут артефакты, которые относятся ко всему проекту: общая концепция, целевая аудитория, словарь терминов, архитектурный обзор. Они видны всем компонентам проекта и переиспользуются между ними.
- **На компоненте** — карточка компонента → вкладка **«Артефакты»**. Здесь — артефакты конкретного компонента: подробные требования, диаграммы потоков, мокапы. Они видны только в рамках своего компонента.

## Какие форматы поддерживаются

Когда вы создаёте новый артефакт, выбираете один из четырёх форматов:

- **Markdown** — основной формат для текста. В нём можно сразу вставлять диаграммы Mermaid (просто кодом внутри Markdown) и подключать картинки.
- **Mermaid** — отдельный артефакт-диаграмма (без обёртки Markdown). Удобно, когда диаграмма самостоятельна и должна жить как отдельный документ.
- **BPMN** — нотация бизнес-процессов. Создаётся в визуальном редакторе с дорожками, шлюзами, событиями.
- **drawio** — произвольное рисование схем: блок-схемы, mind maps, архитектурные диаграммы — всё, что не вписывается в строгие нотации.

При создании можно ввести только название и тип — тело откроется в соответствующем редакторе сразу после нажатия «Создать».


## Как создать артефакт

1. Зайдите на вкладку «Артефакты» проекта или компонента.
2. Нажмите кнопку **«Создать»** (или **«+»**) — откроется диалог.
3. Введите название (что описывает артефакт), выберите формат, при желании сразу напишите тело в Markdown — для остальных форматов оно сгенерируется заготовкой.
4. **Ctrl+Enter** или **⌘+Enter** — отправить.

Артефакт попадает в список вкладки и сразу становится частью проектного контекста: на него можно ссылаться из других артефактов, его текст подтягивается в формирование РИД при сдаче результата.

!!!info "Артефакты на задачах — отдельный путь"
    На некоторых задачах вы можете встретить артефакты, привязанные именно к задаче. Это переходный механизм — со временем такие артефакты переедут в общие списки на компоненте/проекте, чтобы у каждого артефакта было одно место и одна ссылка.


## Как выбрать формат

| Что описываете | Формат |
|---|---|
| Текстовое описание, требования, чек-листы | Markdown |
| Диаграмма последовательности, ER, классы | Markdown с Mermaid внутри (или отдельный Mermaid) |
| Бизнес-процесс с ролями и ветвлениями | BPMN |
| Архитектура / mind map / произвольная схема | drawio |

В сомнительных случаях — Markdown: он самый гибкий, и в нём всегда можно встроить ту же Mermaid-диаграмму как блок кода.


## Артефакты и Результат интеллектуальной деятельности

Когда Мастер Компонента нажимает **«Внести результат»**, бэкенд собирает все артефакты Компонента (плюс артефакты родительского Проекта, плюс описание Проекта, плюс задачи и текст Коммитов) в один документ, считает его SHA-256 и публикует хэш в блокчейне через `capital::pushrslt`. Полный текст хранится во внутренней базе кооператива; в цепочке остаётся только хэш как доказательство существования.

Именно этот собранный документ — и есть **РИД**, который Компонент вносит в кооператив как имущественный паевой взнос. Подробнее — в *Результат интеллектуальной деятельности*.
