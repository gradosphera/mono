# Создание проекта и компонента

Любая работа в Благоросте начинается с проекта и его компонентов.

- **Проект** — направление работы кооператива. Сам по себе паевые взносы не принимает: это шапка над компонентами, объединяющая их по теме.
- **Компонент** — конкретная задача внутри проекта: с описанным результатом, сметой, мастером и командой. Именно к компоненту вступают авторы, исполнители и инвесторы; именно по компоненту совет голосует по приёму результата.

Один проект может содержать произвольное число компонентов.


## Где это и когда вы это делаете

Создание начинается из **Мастерской** — `/<кооператив>/capital/projects`. В правом нижнем углу есть плавающая кнопка «+».

Наведите на неё курсор — она раскрывается и показывает доступные действия. Из Мастерской доступно действие «Проект». Действие «Компонент» появляется внутри проекта — после того, как вы откроете его карточку (или прямо из строки в списке проектов).

![Раскрытая кнопка «+» в Мастерской: действие «Проект»](/assets/new/blagorost/project-create/01-fab-actions.png)

Создавать может **председатель** или **член совета**. У обычного пайщика кнопок «+» нет.


## Создание проекта

Нажмите «+ Проект» (или клавиша **P** на странице Мастерской) — откроется форма.

Введите два поля:

- **Название проекта** — короткое имя, под которым проект будет известен в кооперативе («Кошелёк пайщика», «Платформа Цифрового Кооператива»).
- **Описание проекта** — суть направления: ради чего создаётся, что в него входит, в каком контексте.

![Форма создания проекта «Кошелёк пайщика»](/assets/new/blagorost/project-create/02-create-project-dialog.png)

Нажмите **«Создать»**. Проект появится в списке Мастерской — без компонентов, в статусе «pending».


## Создание компонента

Чтобы добавить компонент в проект — нажмите **«+»** в строке этого проекта (правый край). Откроется форма компонента.

Введите:

- **Название компонента** — то, что описывает результат как отдельную единицу работы («MVP v1», «Версия 1.1», «Минимальный продукт»).
- **Описание компонента** — что именно вы хотите получить, какой результат интеллектуальной деятельности будет считаться выполненным. Это описание потом увидят пайщики, которые захотят к компоненту присоединиться.

![Форма создания компонента «MVP v1» внутри проекта «Кошелёк пайщика»](/assets/new/blagorost/project-create/03-create-component-dialog.png)

Нажмите **«Создать»** — компонент появится внутри своего проекта.


## Что вы получили

В Мастерской теперь видна структура: проект-шапка и под ним компонент. Стрелкой «›» слева от названия проекта раскрывается список его компонентов.

![Проект «Кошелёк пайщика» с компонентом «MVP v1» в Мастерской](/assets/new/blagorost/project-create/04-projects-list-with-koshelek.png)

С этого момента работа идёт уже не с проектом, а с **компонентом**. Дальше — назначение мастера, план, открытие приёма паевых взносов, коммиты исполнителей, голосование совета по приёмке. См. *Назначение мастера и план компонента*.


!!!info "Что не нужно заполнять руками"
    Project hash, parent hash, метаданные — всё это формирует UI автоматически. Для проекта parent пустой; для компонента — он подставляется из проекта, в котором вы нажали «+».
